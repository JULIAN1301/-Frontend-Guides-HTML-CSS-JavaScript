📖 Guía Completa de HTML

Explora todas las etiquetas HTML con ejemplos prácticos y avanzados.

📌 1. Etiquetas de estructura

Estas etiquetas definen la base de una página web y ayudan a organizar el contenido semánticamente.

📜 Lista de etiquetas de estructura

<html>, <head>, <body>, <header>, <footer>, <main>, <section>, <article>, <nav>, 
    <aside>, <div>, <address>, <figure>, <figcaption>, <summary>, <details>, 
    <dialog>, <template>, <fieldset>, <legend>

🌍 1.1. <html> - Documento raíz

La etiqueta <html> encapsula todo el contenido de la página web.

<!DOCTYPE html>
    <html lang="es">
        <head>
            <title>Ejemplo de HTML</title>
        </head>
        <body>
            <h1>Hola, Mundo</h1>
        </body>
    </html>

🧠 1.2. <head> - Información del documento

Contiene metadatos, enlaces a CSS, scripts y el título de la página.

<head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Ejemplo de Head</title>
        <link rel="stylesheet" href="styles.css">
        <script src="script.js"></script>
    </head>

📄 1.3. <body> - Contenido principal

Todo el contenido visible de la página va dentro de esta etiqueta.

<body>
        <h1>Bienvenido a mi página</h1>
        <p>Este es un contenido de ejemplo</p>
    </body>

🔝 1.4. <header> - Encabezado

Define el encabezado de una página o sección.

<header>
        <h1>Mi Blog</h1>
        <nav>
            <a href="#home">Inicio</a>
            <a href="#about">Sobre mí</a>
        </nav>
    </header>

🔚 1.5. <footer> - Pie de página

Ubicado en la parte inferior, contiene información de contacto, derechos de autor, etc.

<footer>
        <p>© 2025 Mi Sitio Web</p>
        <address>Contacto: <a href="mailto:info@misitio.com">info@misitio.com</a></address>
    </footer>

📌 1.6. <main> - Contenido principal

Contiene el contenido más importante de la página.

<main>
        <section>
            <h2>Noticias recientes</h2>
            <article>
                <h3>Título de la noticia</h3>
                <p>Contenido de la noticia...</p>
            </article>
        </section>
    </main>

📌 1.7. <section> - Sección

Agrupa contenido relacionado dentro de la página.

<section>
        <h2>Servicios</h2>
        <p>Ofrecemos desarrollo web y marketing digital.</p>
    </section>

📝 1.8. <article> - Artículo

Se usa para contenido independiente como blogs o noticias.

<article>
        <h2>Cómo aprender HTML</h2>
        <p>HTML es el lenguaje base de la web...</p>
    </article>

🔗 1.9. <nav> - Navegación

Contiene enlaces de navegación.

<nav>
        <a href="#home">Inicio</a>
        <a href="#services">Servicios</a>
    </nav>

📑 1.10. <aside> - Contenido adicional

Ideal para barras laterales con contenido complementario.

<aside>
        <h2>Publicidad</h2>
        <p>Compra ahora y obtén un descuento</p>
    </aside>

📦 1.11. <div> - Contenedor genérico

<div class="contenedor">
        <p>Este es un contenedor sin significado semántico.</p>
    </div>

🖼 1.12. <figure> y <figcaption> - Imágenes con descripción

<figure>
        <img src="imagen.jpg" alt="Descripción de la imagen">
        <figcaption>Esta es la descripción de la imagen</figcaption>
    </figure>

🔍 1.13. <details> y <summary> - Contenido expandible

<details>
        <summary>Haz clic para más información</summary>
        <p>Aquí hay más contenido oculto</p>
    </details>

💬 1.14. <dialog> - Cuadro de diálogo

<dialog id="modal">
        <p>Este es un cuadro de diálogo.</p>
        <button onclick="document.getElementById('modal').close()">Cerrar</button>
    </dialog>
    <button onclick="document.getElementById('modal').showModal()">Abrir diálogo</button>

📄 1.15. <template> - Plantilla reutilizable

<template id="tarjeta">
        <div class="card">
            <h3>Título de la tarjeta</h3>
            <p>Descripción breve.</p>
        </div>
    </template>

⚙️ 3. Atributos en HTML

Los atributos proporcionan información adicional sobre los elementos HTML y modifican su comportamiento.

📜 Lista de atributos comunes


    id, class, style, title, lang, dir, hidden, tabindex, accesskey, draggable, contenteditable,
    src, alt, href, target, download, rel, type, value, placeholder, disabled, readonly, checked,
    selected, multiple, name, action, method, enctype, autocomplete, required, pattern, minlength, maxlength,
    step, autofocus, spellcheck, wrap, cols, rows, size, for, defer, async, crossorigin, charset, 
    http-equiv, name, sandbox, referrerpolicy, loading

🆔 3.1. Identificadores con id

El atributo id asigna un identificador único a un elemento, permitiendo aplicarle estilos y seleccionarlo con JavaScript.

<p id="ejemplo-id">Este párrafo tiene un ID.</p>

Este párrafo tiene un ID.

🎨 3.2. Clases con class

El atributo class permite aplicar estilos a varios elementos.

<p class="ejemplo-clase">Este texto usa una clase.</p>

Este texto usa una clase.

Otro párrafo con la misma clase.

🎭 3.3. Estilos con style

El atributo style define estilos directamente en el elemento.

<p style="color: green; font-size: 20px;">Texto con estilos en línea.</p>

Texto con estilos en línea.

ℹ️ 3.4. Descripción emergente con title

El atributo title muestra un mensaje emergente al pasar el mouse sobre el elemento.

<p title="Este es un tooltip">Pasa el mouse sobre este texto.</p>

Pasa el mouse sobre este texto.

🌍 3.5. Idioma con lang

El atributo lang indica el idioma del contenido.

<p lang="es">Este párrafo está en español.</p>
        <p lang="fr">Ce paragraphe est en français.</p>

Este párrafo está en español.

Ce paragraphe est en français.

📐 3.6. Dirección del texto con dir

El atributo dir controla la dirección del texto.

<p dir="rtl">Este texto se alinea de derecha a izquierda.</p>

Este texto se alinea de derecha a izquierda.

🙈 3.7. Ocultar elementos con hidden

El atributo hidden oculta un elemento sin eliminarlo del DOM.

<p hidden>Este texto está oculto.</p>

Este texto está oculto.

🔢 3.8. Orden de navegación con tabindex

El atributo tabindex define el orden en que los elementos reciben el foco al presionar "Tab".

<input type="text" tabindex="2" placeholder="Segundo campo">
        <input type="text" tabindex="1" placeholder="Primer campo">

⌨️ 3.9. Atajos de teclado con accesskey

El atributo accesskey permite asignar un atajo de teclado.

<p accesskey="h">Presiona "Alt + H" para enfocar este texto.</p>

Presiona "Alt + H" para acceder rápidamente.

🖱️ 3.10. Elementos arrastrables con draggable

El atributo draggable define si un elemento puede ser arrastrado.

<img src="https://via.placeholder.com/100" draggable="true" alt="Imagen arrastrable">

✍️ 3.11. Contenido editable con contenteditable

El atributo contenteditable hace que un elemento sea editable.

<p contenteditable="true">Haz clic aquí y edita este texto.</p>

Haz clic aquí y edita este texto.

🎥 3.12. Ruta de archivos multimedia con src

El atributo src define la ubicación de imágenes, audios y videos.

<img src="https://via.placeholder.com/150" alt="Imagen de prueba">

🖼️ 3.13. Texto alternativo con alt

El atributo alt proporciona una descripción en caso de que la imagen no se cargue.

<img src="imagen-no-existe.jpg" alt="Imagen alternativa">

🔗 3.14. Enlaces con href

El atributo href especifica la dirección a la que apunta un enlace.

<a href="https://www.google.com">Ir a Google</a>
Ir a Google

🌍 3.15. Control de apertura de enlaces con target

El atributo target define cómo se abrirá un enlace (_blank para nueva pestaña).

<a href="https://www.google.com" target="_blank">Abrir en nueva pestaña</a>
Abrir en nueva pestaña

📥 3.16. Descarga de archivos con download

El atributo download permite que el enlace descargue un archivo en lugar de abrirlo.

<a href="https://via.placeholder.com/150" download>Descargar imagen</a>
Descargar imagen

🔗 3.17. Relación entre documentos con rel

El atributo rel define la relación entre la página actual y el enlace.

<a href="https://www.ejemplo.com" rel="noopener noreferrer">Enlace seguro</a>
Enlace seguro

📝 3.18. Tipo de entrada en formularios con type

El atributo type define el tipo de campo en un formulario.

<input type="email" placeholder="Correo electrónico">

📌 3.19. Valor predeterminado en formularios con value

El atributo value define el valor inicial de un campo.

<input type="text" value="Texto por defecto">

💡 3.20. Texto de ayuda con placeholder

El atributo placeholder muestra un texto de ayuda en un campo.

<input type="text" placeholder="Escribe tu nombre">

🚫 3.21. Desactivar un campo con disabled

El atributo disabled hace que el usuario no pueda interactuar con el campo.

<input type="text" disabled value="No editable">

🔒 3.22. Solo lectura con readonly

El atributo readonly hace que el usuario pueda ver pero no modificar el valor.

<input type="text" readonly value="Solo lectura">

✅ 3.23. Casilla marcada por defecto con checked

El atributo checked marca automáticamente un checkbox o un radio button.

<input type="checkbox" checked> Acepto los términos
Acepto los términos

🔘 3.24. Opción seleccionada por defecto con selected

El atributo selected marca una opción por defecto en un select.

<select>
        <option selected>Opción 1</option>
        <option>Opción 2</option>
        </select>

📋 3.25. Selección múltiple con multiple

El atributo multiple permite seleccionar varias opciones en un select.

<select multiple>
        <option>Opción 1</option>
        <option>Opción 2</option>
        </select>

🏷️ 3.26. Nombre de campo con name

El atributo name define el nombre del campo en el formulario.

<input type="text" name="usuario">

📤 3.27. URL de envío con action

El atributo action especifica la URL donde se enviará el formulario.

<form action="procesar.php" method="POST">
        <input type="text" name="usuario">
        <button type="submit">Enviar</button>
        </form>

🔄 3.28. Método de envío con method

El atributo method define si el formulario se envía con GET o POST.

<form method="POST">

📦 3.29. Codificación del formulario con enctype

El atributo enctype define la forma en que se envían los datos en un formulario.

<form enctype="multipart/form-data">

🔍 3.30. Autocompletado con autocomplete

El atributo autocomplete permite que el navegador complete los campos automáticamente.

<input type="text" autocomplete="on">

⚠️ 3.31. Campos obligatorios con required

El atributo required obliga a que el usuario complete el campo antes de enviar el formulario.

<input type="text" required>

🔡 3.32. Validación con patrón usando pattern

El atributo pattern define una expresión regular para validar la entrada del usuario.

<input type="text" pattern="[A-Za-z]{3,}">

📝 3.33. Mínimo de caracteres con minlength

El atributo minlength define la cantidad mínima de caracteres que un usuario debe ingresar.

<input type="text" minlength="5" placeholder="Mínimo 5 caracteres">

🔠 3.34. Máximo de caracteres con maxlength

El atributo maxlength define la cantidad máxima de caracteres permitidos en un campo.

<input type="text" maxlength="10" placeholder="Máximo 10 caracteres">

➕ 3.35. Incrementos numéricos con step

El atributo step define el tamaño del incremento en un campo numérico.

<input type="number" step="5">

🚀 3.36. Autoenfoque con autofocus

El atributo autofocus hace que el campo obtenga el foco automáticamente al cargar la página.

<input type="text" autofocus placeholder="Este campo tiene autoenfoque">

⌨️ 3.37. Corrección ortográfica con spellcheck

El atributo spellcheck activa o desactiva la corrección ortográfica en campos de texto.

<textarea spellcheck="true">Escribe aquí...</textarea>

📜 3.38. Ajuste del texto en textarea con wrap

El atributo wrap controla cómo se ajusta el texto en un `textarea`. - soft: El texto no se ajusta y mantiene el desplazamiento horizontal. - hard: El texto se ajusta automáticamente al ancho del área de texto y al enviar el formulario, se conserva el formato.

<textarea wrap="hard">Texto con ajuste forzado</textarea>

📏 3.39. Columnas en textarea con cols

El atributo cols define el ancho visible en número de caracteres.

<textarea cols="40"></textarea>

📐 3.40. Filas en textarea con rows

El atributo rows define la altura visible en número de líneas de texto.

<textarea rows="5"></textarea>

🔠 3.41. Ancho de entrada con size

El atributo size define el ancho en caracteres de un campo `input type="text"`.

<input type="text" size="20">

🏷️ 3.42. Asociación de etiquetas con for

El atributo for enlaza una etiqueta `label` con un campo `input`, lo que permite que al hacer clic en el texto de la etiqueta, el campo se seleccione automáticamente.

<label for="nombre">Nombre:</label>
        <input type="text" id="nombre">

⚡ 3.43. Carga diferida de scripts con defer

El atributo defer hace que el script se cargue en segundo plano y solo se ejecute después de que el HTML haya sido completamente procesado.

<script src="script.js" defer></script>

🚀 3.44. Carga asíncrona de scripts con async

El atributo async permite que el script se descargue y ejecute en paralelo mientras se procesa el HTML, lo que puede mejorar la velocidad de carga.

<script src="script.js" async></script>

🔗 3.45. Solicitudes entre dominios con crossorigin

El atributo crossorigin se usa para controlar cómo se manejan las solicitudes de recursos externos en imágenes, fuentes o scripts.

<img src="https://via.placeholder.com/150" crossorigin="anonymous" alt="Imagen con crossorigin">

🔤 3.46. Codificación de caracteres con charset

El atributo charset define la codificación de caracteres de la página para soportar caracteres especiales y símbolos correctamente.

<meta charset="UTF-8">

🖥️ 3.47. Cabeceras HTTP con http-equiv

El atributo http-equiv permite enviar cabeceras HTTP desde un documento HTML, como redireccionamientos o configuraciones de caché.

<meta http-equiv="refresh" content="5;url=https://www.ejemplo.com">

📌 3.48. Metadatos con name

El atributo name define información adicional sobre la página web, como su descripción o palabras clave.

<meta name="description" content="Esta es una descripción de la página.">

🔐 3.49. Restricciones en iframe con sandbox

El atributo sandbox restringe las acciones que un `iframe` puede realizar, evitando ejecución de scripts o accesos no autorizados.

<iframe src="https://www.example.com" sandbox></iframe>

🔍 3.50. Política de referencia con referrerpolicy

El atributo referrerpolicy controla qué información de referencia se envía al abrir enlaces externos, mejorando la privacidad.

<a href="https://www.ejemplo.com" referrerpolicy="no-referrer">Enlace sin referencia</a>
Enlace sin referencia

📷 3.51. Carga diferida de imágenes con loading

El atributo loading optimiza la carga de imágenes al diferirlas hasta que estén a punto de aparecer en pantalla.

<img src="https://via.placeholder.com/150" loading="lazy" alt="Imagen con carga diferida">

🔗 4. Enlaces

Estas etiquetas permiten la navegación dentro de una página web o hacia otras páginas.

📜 Lista de etiquetas de enlaces

<a>, <nav>, <map>, <area>

Visitar Google

🔗 4.3. Enlace con atributo title

Ejemplo:

Ir a Wikipedia 
<a href="https://www.wikipedia.org" title="Visita Wikipedia">Ir a Wikipedia</a>

🔗 4.4. Enlace que abre en nueva pestaña

Ejemplo:

Ir a Mozilla 
<a href="https://www.mozilla.org" target="_blank" rel="noopener noreferrer">Ir a Mozilla</a>

📥 4.5. Enlace de descarga

Ejemplo:

Descargar documento PDF

📧 4.6. Enlace a correo electrónico

Ejemplo:

Enviar correo 
<a href="mailto:correo@ejemplo.com">Enviar correo</a>

📞 4.7. Enlace a número de teléfono

Ejemplo:

Llamar al +123456789

🧭 4.8. Menú de navegación con <nav>

Ejemplo:

  • Inicio
  • Nosotros
  • Contacto
<nav>
        <ul>
            <li><a href="#inicio">Inicio</a></li>
            <li><a href="#nosotros">Nosotros</a></li>
            <li><a href="#contacto">Contacto</a></li>
        </ul>
    </nav>

📌 4.9. Mapa de imagen con <map> y <area>

Ejemplo:

<img src="Google_logo.svg.png" usemap="#mapaEjemplo" alt="Mapa de imagen de Google" width="300">
    <map name="mapaEjemplo">
        <area shape="rect" coords="0,0,150,150" href="https://www.google.com" alt="Google">
        <area shape="rect" coords="150,0,300,150" href="https://www.wikipedia.org" alt="Wikipedia">
    </map>

📋 5. Listas

Las listas en HTML permiten organizar información en formatos ordenados, desordenados y de definiciones.

📜 Lista de etiquetas de listas

<ul>, <ol>, <li>, <dl>, <dt>, <dd>

✅ 5.1. Lista desordenada <ul>

Se usa para listar elementos sin un orden específico.

  • Manzana
  • Banana
  • Naranja
<ul>
        <li>Manzana</li>
        <li>Banana</li>
        <li>Naranja</li>
    </ul>

🔢 5.2. Lista ordenada <ol>

Se usa cuando el orden de los elementos es importante.

  1. Encender la computadora
  2. Abrir el navegador
  3. Visitar Google
<ol>
        <li>Encender la computadora</li>
        <li>Abrir el navegador</li>
        <li>Visitar Google</li>
    </ol>

📌 5.3. Lista de definición <dl>

Se usa para definir términos con sus respectivas descripciones.

HTML
Lenguaje de marcado para la web.
CSS
Hojas de estilo para el diseño web.
JavaScript
Lenguaje de programación para la interactividad en la web.
<dl>
        <dt>HTML</dt>
        <dd>Lenguaje de marcado para la web.</dd>
        <dt>CSS</dt>
        <dd>Hojas de estilo para el diseño web.</dd>
        <dt>JavaScript</dt>
        <dd>Lenguaje de programación para la interactividad en la web.</dd>
    </dl>

🎨 5.4. Lista ordenada con tipo personalizado

Podemos personalizar el tipo de numeración con el atributo type.

  1. Primer punto
  2. Segundo punto
  3. Tercer punto
<ol type="I">
        <li>Primer punto</li>
        <li>Segundo punto</li>
        <li>Tercer punto</li>
    </ol>

📌 5.5. Lista anidada

Podemos crear listas dentro de listas.

  • Frutas
    • Manzana
    • Banana
    • Naranja
  • Verduras
    • Zanahoria
    • Lechuga
    • Brócoli
<ul>
        <li>Frutas
            <ul>
                <li>Manzana</li>
                <li>Banana</li>
                <li>Naranja</li>
            </ul>
        </li>
        <li>Verduras

                <li>Zanahoria</li>

        </li>

📝 7. Formularios

Los formularios permiten recopilar información del usuario mediante diferentes elementos de entrada.

📜 Lista de etiquetas de formularios

<form>, <input>, <textarea>, <button>, <select>, <option>, <label>, 
    <fieldset>, <legend>, <datalist>, <optgroup>, <output>, <progress>, <meter>

📝 7.1. Formulario básico

Un formulario simple con un campo de texto y un botón de envío.

<form>
        <label for="nombre">Nombre:</label>
        <input type="text" id="nombre" name="nombre" placeholder="Escribe tu nombre">
        <button type="submit">Enviar</button>
    </form>

📌 7.2. Diferentes tipos de <input>

El elemento <input> tiene varios tipos para diferentes entradas de usuario.








<form>
        <label>Email: <input type="email" placeholder="ejemplo@correo.com"></label><br>
        <label>Contraseña: <input type="password"></label><br>
        <label>Número: <input type="number" min="1" max="10"></label><br>
        <label>Fecha: <input type="date"></label><br>
        <label>Archivo: <input type="file"></label><br>
        <label>Color: <input type="color"></label><br>
        <label>Rango: <input type="range" min="0" max="100"></label><br>
        <button type="submit">Enviar</button>
    </form>

📌 7.3. Área de texto con <textarea>

Para escribir comentarios o mensajes largos.



<form>
        <label for="mensaje">Mensaje:</label><br>
        <textarea id="mensaje" name="mensaje" rows="4" cols="40" placeholder="Escribe tu mensaje..."></textarea><br>
        <button type="submit">Enviar</button>
    </form>

📌 7.4. Lista desplegable con <select> y <option>

Se pueden crear listas desplegables con varias opciones.

<form>
        <label for="pais">Selecciona tu país:</label>
        <select id="pais" name="pais">
            <option value="mx">México</option>
            <option value="es">España</option>
            <option value="ar">Argentina</option>
        </select>
        <button type="submit">Enviar</button>
    </form>

📌 7.5. Agrupar opciones con <optgroup>

Las listas desplegables pueden tener grupos de opciones.

<form>
        <label for="ciudad">Selecciona tu ciudad:</label>
        <select id="ciudad" name="ciudad">
            <optgroup label="México">
                <option value="cdmx">Ciudad de México</option>
                <option value="gdl">Guadalajara</option>
            </optgroup>
            <optgroup label="España">
                <option value="mad">Madrid</option>
                <option value="bcn">Barcelona</option>
            </optgroup>
        </select>
        <button type="submit">Enviar</button>
    </form>

📌 7.6. Salida de datos con <output>

El elemento <output> muestra valores calculados.

+ = 0 
<form oninput="resultado.value = parseInt(num1.value) + parseInt(num2.value)">
        <input type="number" id="num1" name="num1" value="0"> +
        <input type="number" id="num2" name="num2" value="0"> =
        <output name="resultado">0</output>
    </form>

🎬 8. Multimedia

Las etiquetas multimedia permiten incluir contenido audiovisual e imágenes en una página web.

📜 Lista de etiquetas multimedia

<audio>, <video>, <picture>, <source>, <track>, <embed>, <iframe>, <object>, <img>, <figure>, <figcaption>, <map>, <area>

🎵 8.1. Audio con <audio>

Para incluir archivos de sonido en una página.

<audio controls>
        <source src="audio.mp3" type="audio/mpeg">
        <source src="audio.ogg" type="audio/ogg">
        Tu navegador no soporta la reproducción de audio.
    </audio>

🎥 8.2. Video con <video>

Para incrustar videos con diferentes formatos.

<video controls width="400">
        <source src="video.mp4" type="video/mp4">
        <source src="video.ogg" type="video/ogg">
        Tu navegador no soporta la reproducción de video.
    </video>

🖼️ 8.3. Imágenes con <img>

Permite mostrar imágenes en una página.

<img src="Google_logo.svg.png" alt="Logo de Google" width="150">

🖼️ 8.4. Imágenes adaptativas con <picture>

Permite cargar diferentes imágenes según el tamaño de pantalla.

<picture>
        <source srcset="imagen_grande.jpg" media="(min-width: 800px)">
        <source srcset="imagen_pequena.jpg" media="(max-width: 799px)">
        <img src="imagen_default.jpg" alt="Imagen adaptable">
    </picture>

🖼️ 8.5. Imagen con descripción <figure> y <figcaption>

Ejemplo:

<figure>
        <img src="Google_logo.svg.png" alt="Logo de Google con descripción" width="200">
        <figcaption>El logo de Google, una de las empresas más grandes del mundo.</figcaption>
    </figure>

📌 8.6. Mapa de imagen con <map> y <area>

Ejemplo:

<img src="Google_logo.svg.png" usemap="#mapaEjemplo" alt="Mapa de imagen de Google" width="300">
    <map name="mapaEjemplo">
        <area shape="rect" coords="0,0,150,150" href="https://www.google.com" alt="Google">
        <area shape="rect" coords="150,0,300,150" href="https://www.wikipedia.org" alt="Wikipedia">
    </map>

🎭 8.7. Subtítulos en videos con <track>

Se usa para agregar subtítulos en distintos idiomas.

<video controls width="400">
        <source src="video.mp4" type="video/mp4">
        <track src="subtitulos_es.vtt" kind="subtitles" srclang="es" label="Español">
        <track src="subtitulos_en.vtt" kind="subtitles" srclang="en" label="English">
    </video>

🌍 8.8. Incrustar contenido externo con <iframe>

Ejemplo de un video de YouTube incrustado.

<iframe width="560" height="315" 
        src="https://www.youtube.com/embed/kN1XP-Bef7w" 
        title="Video de ejemplo" frameborder="0" allowfullscreen>
    </iframe>

📂 8.9. Contenido incrustado con <embed>

Ejemplo de un contenido incrustado.

<embed src="documento.pdf" type="application/pdf" width="600" height="450">

📜 8.10. Objetos incrustados con <object>

Ejemplo de un PDF incrustado.

<object data="documento.pdf" type="application/pdf" width="600" height="400">
        Tu navegador no soporta la visualización de PDFs.
    </object>

📝 10. Metaetiquetas en HTML

Las metaetiquetas proporcionan información sobre la página web a los navegadores y motores de búsqueda.

📜 Lista de etiquetas de estructura

Estas etiquetas ayudan a estructurar correctamente el contenido de una página web.

<html>, <head>, <body>, <header>, <footer>, <main>, <section>, <article>, <nav>, 
        <aside>, <div>, <address>, <figure>, <figcaption>, <summary>, <details>, 
        <dialog>, <template>, <fieldset>, <legend>

🔠 10.1. Definir la codificación de caracteres con charset

El atributo charset define la codificación de caracteres de la página para evitar errores con acentos y caracteres especiales.

<meta charset="UTF-8">

📌 10.2. Especificar descripción con name="description"

El atributo name="description" proporciona un resumen del contenido de la página, útil para SEO.

<meta name="description" content="Guía completa sobre HTML y sus etiquetas.">

🔍 10.3. Palabras clave con name="keywords"

El atributo name="keywords" define palabras clave relacionadas con el contenido de la página.

<meta name="keywords" content="HTML, CSS, JavaScript, desarrollo web">

🌍 10.4. Configurar el viewport con name="viewport"

El atributo name="viewport" define cómo se muestra la página en dispositivos móviles.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

🔄 10.5. Redireccionamiento con http-equiv="refresh"

El atributo http-equiv="refresh" permite redirigir a otra página después de un tiempo determinado.

<meta http-equiv="refresh" content="5;url=https://www.ejemplo.com">

⏳ 10.6. Control de caché con http-equiv="cache-control"

El atributo http-equiv="cache-control" indica cómo se debe almacenar la página en caché.

<meta http-equiv="cache-control" content="no-cache">

🔗 10.7. Política de referencia con referrerpolicy

El atributo referrerpolicy define qué información de referencia se envía al abrir enlaces externos.

<meta name="referrer" content="no-referrer">

🛑 10.8. Configuración de X-UA-Compatible

El atributo http-equiv="X-UA-Compatible" indica cómo se debe procesar la página en navegadores antiguos.

<meta http-equiv="X-UA-Compatible" content="IE=edge">

🔏 10.9. Seguridad con http-equiv="content-security-policy"

El atributo http-equiv="content-security-policy" ayuda a prevenir ataques como inyección de scripts.

<meta http-equiv="content-security-policy" content="default-src 'self'">

📲 10.10. Icono de la página con link rel="icon"

Para definir un favicon, se usa la etiqueta link con rel="icon".

<link rel="icon" href="favicon.ico" type="image/x-icon">

♿ 11. Accesibilidad en HTML

La accesibilidad web garantiza que las personas con discapacidad puedan navegar e interactuar con los sitios web de manera efectiva. HTML proporciona diversas etiquetas y atributos para mejorar la accesibilidad.

📜 Lista de etiquetas y atributos de accesibilidad

<label>, <alt>, <title>, <aria-label>, <aria-labelledby>, <aria-describedby>, <role>, 
        <tabindex>, <accesskey>, <aria-hidden>, <aria-live>, <aria-disabled>, <aria-expanded>, 
        <aria-checked>, <aria-pressed>, <aria-selected>, <aria-busy>

🏷️ 11.1. Etiquetas de formularios con label

El uso de label mejora la accesibilidad de los formularios al enlazar etiquetas con campos de entrada.

<label for="email">Correo electrónico:</label>
        <input type="email" id="email" name="email">

🖼️ 11.2. Texto alternativo con alt

El atributo alt proporciona una descripción textual para imágenes, útil para lectores de pantalla.

<img src="imagen.jpg" alt="Descripción de la imagen">

ℹ️ 11.3. Información adicional con title

El atributo title proporciona información adicional cuando el usuario pasa el cursor sobre un elemento.

<p title="Este es un mensaje de ayuda">Pasa el mouse sobre este texto.</p>

Pasa el mouse sobre este texto.

🔠 11.4. Navegación con tabindex

El atributo tabindex controla el orden de navegación con la tecla "Tab".

<input type="text" tabindex="2" placeholder="Segundo campo">
        <input type="text" tabindex="1" placeholder="Primer campo">

⌨️ 11.5. Atajos de teclado con accesskey

El atributo accesskey permite acceder rápidamente a un elemento mediante combinaciones de teclas.

<p accesskey="h">Presiona "Alt + H" para acceder rápido.</p>

Presiona "Alt + H" para acceder rápido.

🔒 11.6. Elementos ocultos con aria-hidden

El atributo aria-hidden="true" oculta un elemento para lectores de pantalla sin ocultarlo visualmente.

<p aria-hidden="true">Este texto es invisible para lectores de pantalla.</p>

Este texto es invisible para lectores de pantalla.

📢 11.7. Áreas en vivo con aria-live

El atributo aria-live indica que el contenido del elemento puede cambiar dinámicamente y debe ser anunciado por lectores de pantalla.

<div aria-live="polite">Mensaje importante que se actualizará.</div>
Mensaje importante que se actualizará.

🛑 11.8. Deshabilitar elementos con aria-disabled

El atributo aria-disabled="true" informa a los lectores de pantalla que el elemento está deshabilitado.

<button aria-disabled="true">Botón deshabilitado</button>

🔼 11.9. Elementos expandibles con aria-expanded

El atributo aria-expanded indica si un elemento, como un menú desplegable, está expandido o colapsado.

<button aria-expanded="false">Mostrar opciones</button>

✅ 11.10. Estados de selección con aria-checked y aria-selected

El atributo aria-checked indica si un checkbox está marcado, y aria-selected si un elemento de una lista está seleccionado.

<input type="checkbox" aria-checked="true"> Opción activada
Opción activada
